# 全栈教学流水线 Skill 设计文档

**日期**: 2026-06-01
**状态**: 已确认

---

## Context

田总正在创建一个 7 阶段的全栈学习项目（`/Users/Roger/全栈学习/`）。当前内容聚焦 Next.js，但 Skill 本身应是通用的教学流水线，适用于任何技术栈的教程生成。阶段 0 已手动生成，质量满意。现在需要将生成流程标准化为一个 Claude Code Skill，确保后续阶段内容质量一致、流程可追溯、日志可复盘。

## 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 教学风格 | 延续阶段 0：一个概念一个文件 | 田总确认满意 |
| 质量评估 | 自动 5 维度评分 + 修订循环 | 减少人工审阅负担 |
| 日志格式 | Markdown 文件 | 可读性好，方便查看 |
| 配套练习 | 概念级问题 + 阶段级综合练习 | 平衡深度和可操作性 |
| 流水线触发 | 全自动一键生成 | 效率优先 |
| Skill 架构 | 方案 B：SKILL.md + 独立参考文件 | 职责分离，便于维护 |
| 通用性 | 技术栈无关的通用流水线 | 未来可复用于其他教学项目 |

## Skill 文件结构

```
.claude/skills/tutorial-pipeline/
├── SKILL.md                    # 主控流程（技术栈无关）
├── quality-rubric.md           # 质量评分标准（5 维度）
├── content-template.md         # 教学内容文件模板
└── log-template.md             # 生成日志格式模板
```

## SKILL.md 主控流水线

每个阶段执行 8 步：

```
1. 读取阶段定义 → 从 SYLLABUS.md 获取概念清单 + 深度评分
2. 逐概念生成内容 → 按 content-template.md 模板生成
3. 质量评分 → 按 quality-rubric.md 5 维度打分（每个概念）
4. 修订循环 → 均分 < 4.0 自动修订，最多 2 轮
5. 阶段级练习 → 综合练习 + 验证问题
6. 更新 README → 标记已完成概念
7. 记录日志 → 按 log-template.md 追加到 logs/generation-log.md
8. Git 提交 → 每阶段独立 commit
```

已完成的阶段自动跳过（检查 README 中的 checkbox 状态）。

全自动模式：一次性跑完所有未完成阶段。

## 质量评分体系

5 维度，每维度 1-5 分，均分 >= 4.0 通过：

1. **概念清晰度** — 核心概念是否用一句话精准概括
2. **类比有效性** — 类比是否精准、生活化、零基础可理解
3. **代码准确性** — 代码正确、有上下文、有 AI 线索标记
4. **前置依赖连贯** — 严格按依赖链，每个新概念有铺垫
5. **深度匹配度** — 精准匹配 SYLLABUS.md 的深度评分

修订规则：
- 均分 < 4.0 → 自动修订，标注具体低分维度
- 单维度 1 分 → 无论均分必须修订
- 最多 2 轮修订，仍不达标 → 日志标记需人工审阅

## 内容模板

每个概念文件结构：
1. **一句话** — 不超过 30 字核心概括
2. **为什么需要它** — 从学员视角解释痛点
3. **类比** — 生活化类比（表格形式）
4. **核心内容** — 讲解 + 代码示例
5. **你需要记住的** — 3-5 条要点
6. **AI 代码中的线索** — 如何在 AI 代码中识别此概念
7. **验证问题** — 2-3 道理解检查题

## 日志格式

每次生成追加到 `logs/generation-log.md`，包含：
- 生成概况（阶段号、概念数、修订轮次）
- 质量评分表（每个概念 5 维度分数）
- 关键问题（修订了什么、为什么）
- 阶段练习信息

## 触发规则

- "开始阶段 X" / "生成阶段 X" → 单阶段生成
- "生成所有阶段" / "一键生成" → 全自动跑完所有未完成阶段
- "检查质量" → 对已生成内容重新评分

## 验证方法

1. 执行 "生成阶段 1" 验证单阶段流水线
2. 检查生成的文件是否符合 content-template.md
3. 检查 logs/generation-log.md 日志记录
4. 检查 git commit 历史
5. 对阶段 1 内容执行 "检查质量" 验证评分系统
