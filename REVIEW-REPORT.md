## 二、详细问题分析

### 问题组1：React 核心机制缺失（严重）

**现状**：
- 阶段 1.1 提到 JSX 编译成 `React.createElement`，但没讲为什么需要这一层
- 提到 `key` 属性帮助 React 识别变化，但没讲 diff 算法
- 提到组件树，但没讲虚拟DOM是什么

**影响**：学员知道"怎么用"，不知道"为什么这样设计"
**建议**：新增阶段 1.8 "React 渲染机制"，或扩充 1.1

---

### 问题组2：Prisma Schema 属性未讲解（中等）

**现状**：
```prisma
model User {
  id String @id @default(uuid())  # 这三个装饰器只列举，不解释
  email String @unique
}
```

**影响**：学员能模仿写 Schema，但不知道每个装饰器的含义和约束
**建议**：在 5.3 中增加"Prisma Schema 装饰器详解"子章节

---

### 问题组3：水合（Hydration）缺失（中等）

**现状**：
- 阶段 3.1 提到"Server Components + SSR"，但没解释水合过程
- 学员可能困惑：HTML 返回了，JS 怎么绑定事件？

**影响**：不理解 Next.js 的完整渲染流程
**建议**：在阶段 3.1 或 3.4 中增加"水合过程"图解

---

### 问题组4：三种数据获取方式未对比（严重）

**现状**：
- 阶段 1.3 讲 `useEffect` + `fetch`（客户端获取）
- 阶段 3.4 讲 Server Components 直接 `await`（服务端获取）
- 但没有系统对比三种方式的适用场景

**影响**：学员不知道何时用哪种方式
**建议**：新增阶段 3.5 "三种数据获取方式对比"
