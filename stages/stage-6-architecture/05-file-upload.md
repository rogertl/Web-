# 6.5 文件上传与存储

## 一句话

文件上传用 `<input type="file">` 和 FormData，存储到 Vercel Blob、AWS S3 或 Cloudflare R2，用 URL 保存到数据库。

## 为什么需要它

文件不能直接存数据库（太大、太慢），需要专门的存储服务。存储服务提供 CDN 加速、备份、压缩等功能，让文件访问更快更安全。

## 类比

把文件上传想象成"行李寄存"：

| 概念 | 类比 |
|------|------|
| 文件上传 | 寄存行李（把包交给寄存处） |
| 存储服务 | 寄存柜（专门存放行李的地方） |
| 文件 URL | 寄存票据（凭票据取行李） |
| 数据库记录 | 行李登记（登记"几号柜是什么"） |

## 核心内容

| 存储服务 | 特点 |
|---------|------|
| Vercel Blob | Next.js 团队出品，集成简单 |
| AWS S3 | 行业标准，功能强大 |
| Cloudflare R2 | 免费出口流量，性价比高 |

### 上传流程

1. 前端：用户选择文件
2. 前端：FormData 包装文件
3. 服务端：接收文件并上传到存储
4. 服务端：获取文件 URL
5. 服务端：保存 URL 到数据库

## 你需要记住的

1. 文件上传用 FormData 和 `<input type="file">`。
2. 存储到 Vercel Blob、AWS S3 或 Cloudflare R2。
3. 数据库只存文件 URL，不存文件内容。
4. 返回文件 URL 供前端显示或下载。
