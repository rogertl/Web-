# 阶段 1 综合练习

## 练习目标

通过一个完整的 Todo 应用，综合运用阶段 1 的所有核心概念：
- React 组件与 JSX
- Props 组件传值
- Hooks 状态管理
- 组件通信与状态提升
- React Router 路由
- 懒加载与 Suspense
- Client Component 交互

## 练习内容

让 AI 生成以下功能的 Todo 应用代码：

### 功能需求

1. **Todo 列表展示**
   - 显示所有 Todo 项
   - 每个 Todo 有文本和完成状态
   - 可以标记完成/未完成
   - 可以删除 Todo

2. **Todo 创建**
   - 输入框输入新 Todo 文本
   - 点击"添加"按钮创建
   - 按 Enter 键也可以创建

3. **过滤功能**
   - 显示全部/已完成/未完成的 Todo
   - 过滤状态通过 URL 参数管理（如 `?filter=completed`）

4. **路由导航**
   - 首页：Todo 列表
   - 设置页：应用设置（如主题切换）
   - 使用懒加载加载设置页

5. **交互反馈**
   - 添加 Todo 时显示成功提示
   - 删除时显示确认动画

### 组件结构要求

```
App (Client Component)
├── TodoList (展示列表)
│   └── TodoItem (单个 Todo)
├── TodoForm (添加表单)
├── FilterBar (过滤按钮)
└── Settings (设置页，懒加载)
```

## 练习步骤

### 1. 让 AI 生成代码

向 AI 提出以下请求：

```
请帮我生成一个 React Todo 应用，要求：
1. 使用函数组件和 TypeScript
2. 用 useState 管理 Todo 状态和过滤状态
3. 组件之间通过 Props 传递数据和回调函数
4. 使用 React Router 实现首页和设置页的导航
5. 设置页使用 lazy 懒加载
6. 过滤状态通过 URL searchParams 管理
7. 所有交互逻辑用 Client Component 实现

请生成完整的代码，包括所有组件文件。
```

### 2. 用自己的话讲解

拿到代码后，不看任何文档，用自己的话回答以下问题：

**数据流动：**
- 用户在输入框输入文本并点击"添加"后，数据是如何从子组件流向父组件，最终更新 Todo 列表的？
- 过滤按钮点击后，URL 如何变化？组件如何监听这个变化并更新显示的列表？

**组件关系：**
- 哪些组件需要标记 `'use client'`？为什么？
- 为什么某些状态需要提升到父组件，而不是在子组件管理？
- Props drilling 体现在哪里？有没有优化的空间？

**路由和性能：**
- React Router 是如何实现页面切换而不刷新浏览器的？
- 设置页懒加载是如何工作的？Suspense 的 fallback 什么时候显示？

**代码线索：**
- 在 AI 生成的代码中，找出所有阶段 1 讲到的概念（组件、Props、Hooks、路由等）的具体代码模式。

### 3. 修改验证

在 AI 生成的基础上，尝试以下修改：

1. 添加一个新的过滤条件：`?filter=urgent`（只显示包含"紧急"文字的 Todo）
2. 给 TodoItem 添加一个"编辑"功能，点击后可以修改 Todo 文本
3. 把 Settings 组件的懒加载改成普通 import，观察 bundle 大小的变化

如果无法完成这些修改，说明对应的概念还未完全掌握，回到对应章节复习。

## 达标标准

完成此练习后，你应该能够：

- [ ] 看懂 AI 生成的中型 React 项目代码结构
- [ ] 清楚地解释数据在组件之间如何流动
- [ ] 识别代码中哪些部分用到了阶段 1 的哪些概念
- [ ] 独立修改代码添加新功能，不需要反复查阅文档
