# AI 时代现代 React + Next.js 全栈代码理解

> **不是教你写代码，而是教你读懂 AI 生成的代码**

---

## 项目定位

当你让 AI 生成了一个 Next.js 全栈项目，代码能跑，但你不知道：
- `npm run dev` 背后发生了什么？
- Server Component 和 Client Component 为什么要分开？
- Server Actions 和 API 路由有什么区别？
- AI 为什么要这样写，而不是那样写？

**本项目不教你从零写代码**，而是教你：
1. **理解 AI 代码的结构和数据流动**
2. **识别常见模式和最佳实践**
3. **遇到问题时知道去哪里找答案**

## AI 时代的学习方法

### 传统学习路径 vs AI 时代学习路径

| 维度 | 传统学习 | AI 时代学习 |
|------|---------|-------------|
| 起点 | 语法、API、基础概念 | **实际项目代码** |
| 方式 | 从简单到复杂，线性积累 | **从问题到原理，按需深入** |
| 重点 | 记忆 API、练习写代码 | **理解意图、识别模式** |
| 目标 | 能独立实现功能 | **能读懂和修改 AI 代码** |
| 工具 | IDE、文档、Google | **AI + 文档 + 精准搜索** |

### 核心区别

**传统学习**：
```
学语法 → 学框架 → 做项目 → 遇到问题 → 查文档
```

**AI 时代学习**：
```
看 AI 代码 → 理解意图 → 识别模式 → 按需查原理 → 修改优化
```

**本质变化**：
- 从"积累知识"到"运用判断力"
- 从"记住怎么做"到"理解为什么"
- 从"线性学习"到"问题驱动学习"

## 如何使用本项目

### 学习路径

```
1. 按阶段顺序学习（0 → 1 → 2 → 3 → 4 → 5 → 6）
   每个阶段结束后，让 AI 生成对应的小功能

2. 用自己的话讲解数据流
   "这个代码在干什么？数据从哪里来？到哪里去？"

3. 验证理解程度
   能看懂 AI 生成的中型项目代码（Server Action + 数据库 + 权限）
   就算本阶段达标
```

### 典型学习场景

**场景 1：看 AI 生成的代码**
```
你看到这段代码：
export default function UserList() {
  const [users, setUsers] = useState([])

  useEffect(() => {
    fetch('/api/users').then(res => res.json()).then(setUsers)
  }, [])

  return <div>...</div>
}
```

**以前**：不知道 useState、useEffect 是什么，懵了

**现在**：
- 认出 Client Component 交互模式（阶段 1.7）
- 知道 useEffect 是副作用钩子（阶段 1.3）
- 理解数据获取为什么在 useEffect 里（阶段 3）
- 能判断这段代码是否有更好的写法
```

**场景 2：修改 AI 代码**
```
你想添加一个"加载中"状态

以前：试了好几种写法，不知道哪个对

现在：
- 知道加载状态是 Client Component 的常见模式（阶段 1.7）
- 知道 useState 声明状态，useEffect 更新状态（阶段 1.3）
- 知道 Suspense 可以处理异步加载（阶段 1.6）
- 能选择最合适的方案
```

## 核心设计原则

### 1. 概念闭环

**问题**：传统教程经常提到未解释的概念，导致"跳跃式理解"

**解决**：所有技术概念都在 `CONCEPTS.md` 中注册
- 首次出现：简要解释 + 标注后续章节
- 正式讲解：独立段落完整讲解
- 状态跟踪：✅ 已讲解 | ⏳ 待生成 | ❌ 缺失

### 2. 原理优先

**问题**：只讲"怎么用"，不讲"为什么"

**解决**：每个概念都回答三个问题
- 为什么需要它？（解决了什么问题）
- 本质是什么？（底层原理）
- 怎么判断用得好坏？（最佳实践）

### 3. 生活化类比

**问题**：技术解释太抽象，难以理解

**解决**：每个概念都有类比表格

| 技术概念 | 生活类比 |
|---------|---------|
| Server Component | 餐厅后厨——提前做好菜，上菜快 |
| Client Component | 餐厅餐桌——现场拼装，灵活可变 |

### 4. 质量守门

**问题**：教程质量参差不齐，不知道学完是否达标

**解决**：5 维度质量评分
1. 概念清晰度（能否一句话说清楚）
2. 类比有效性（类比是否贴切）
3. 代码准确性（代码是否正确可运行）
4. 前置依赖连贯（是否依赖前文概念）
5. 深度匹配度（深度评分是否符合预期）

**不达标内容自动修订**，最多 2 轮。
